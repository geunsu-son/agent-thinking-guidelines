요약: reg_dt는 등록일로 해석. 결제일과 혼동 시 기수별 집계가 어긋남.

## 배경
2026-03 분석에서 reg_dt를 결제일로 가정해 27기 등록 수가 15% 과소 집계됨.
사용자 확인 후 등록일 컬럼으로 수정.

## 규칙
- reg_dt는 **등록일**로 처리한다.
- 결제일이 필요하면 별도 컬럼(paid_at 등)을 확인하고, 없으면 질문한다.
- 기수별 집계 전 reg_dt NULL 비율과 최소/최대 날짜를 로그로 남긴다.

## SSOT와의 관계
`docs/에이전트_사고행동_지침.md` 1.2(모호함 처리)의 판단 예시와 동일 주제. 충돌 시 SSOT가 우선.
