요약: created_at은 생성일로 해석. 수정일과 혼동 시 월별 집계가 어긋남.

## 배경
2026-03 분석에서 created_at을 수정일로 가정해 이번 달 신규 사용자 수가 15% 과소 집계됨.
사용자 확인 후 생성일 컬럼으로 수정.

## 규칙
- created_at은 **생성일**로 처리한다.
- 수정일이 필요하면 updated_at 등 별도 컬럼을 확인하고, 없으면 질문한다.
- 월별 집계 전 created_at NULL 비율과 최소/최대 날짜를 로그로 남긴다.

`docs/agent-thinking-guidelines.md` 1.2(모호함 처리)의 판단 예시와 동일 주제. 충돌 시 SSOT가 우선.
