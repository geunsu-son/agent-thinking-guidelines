# 세션 간 기억 (메모리)

이 디렉터리에 프로젝트별 교훈을 파일로 축적한다. SSOT는 `docs/agent-thinking-guidelines.md` 8장.

## 규칙
- **한 교훈 = 한 파일.** 파일명은 주제 식별자 (예: `sql-date-filter.md`).
- 맨 위 한 줄: `요약: …`
- 같은 주제는 새 파일 대신 기존 파일 수정. 틀린 메모는 삭제.

## 부트스트랩
1. 설치 시 이 디렉터리를 프로젝트 `.claude/memory/`에 복사한다 (비어 있어도 됨).
2. 형식 참고: `sql-date-filter.example.md` (`.example.md`는 템플릿 repo용 예시, 대상 프로젝트에 복사할 필요 없음).
3. 세션 종료 전, 새 교훈이 있으면 메모 파일로 남긴다.
4. 다음 세션 시작 시 `memory/`를 훑되 **각 파일의 `요약:` 한 줄만 먼저 읽고**, 지금 작업과 관련된 주제만 본문을 연다. (파일이 쌓여도 스캔 비용을 낮게 유지)
